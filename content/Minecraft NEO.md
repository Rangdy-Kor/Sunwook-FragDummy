# Minecraft NEO

---

- 개발 언어
  - 코어 엔진 / 물리 연산 / 그래픽 API → C++ (스마트 포인터 적극 사용)
  - 인게임 컨텐츠 / 고차원 로직 → C# (.NET 10 사용)
  - C++ - C# 연동 → `[LibraryImport]` / 함수 포인터
- 그래픽 API
  - 추상화 → Bgfx
  - Windows / Xbox → DirectX 12
  - Linux / Android → Vulkan
  - macOS / iPadOS / iOS → Metal
  - Nintendo Switch → NVN
  - Play Station → AGC
- 배포
  - 게임 본체 → Native AOT
  - 모딩 레이어 → JIT
  - Native AOT - JIT 연동 → `nethost.dll` 임베디드 런타임
  - 리플렉션 지원 제거 → `rd.xml` 미포함
- 모딩
  - 공식 SDK → Minecraft .NET SDK
    - 모드 간 어셈블리 격리
    - 모드 별 권한 제어
    - NuGet 지원
    - Visual Studio 템플릿
  - 공식 API → MSOM (MicroSoft Open Modding) API
    - Event Bus 시스템
    - 추상화 레이어
  - 공식 확장 프로그램 → Minecraft Scripting Extension
    - TypeScript 지원
    - VS Code 확장 프로그램
  - 모딩 범위
    - C++ → X (All)
    - C# Moding → O (Desktop) | X (Mobile)
      - DLL 동적 로드
      - Sandbox 실행 및 API 통신 (Host 침범 방지)
    - TypeScript Scripting → O (All)
      - Chakra JS Runtime 엔진
- 인게임 명령어
  - MineShell → Java Edition 기반 + Bedrock Edition 전용 명령어 흡수
  - Mine PowerShell → 서버 콘솔 / 데이터팩 한정
    - PowerShell 7 명령 구조
    - .NET 10 기반
    - NEO Edition 전용 개발 언어 (기존 PowerShell과 완전히 별개)
- 서비스 정책
  - 번역 → 현지 유저 투표 체계
  - 공식 콘텐츠 유통 경로 → 마켓플레이스
    - Mine Coin 사용 (Bedrock Edition 계정 소유 코인 연동)
    - 콘텐츠 업로드 시 공식 운영진 감사
    - 라이선스 자유
    - 수수료 존재
  - 외부 유통 → EULA 조건부 허용
    - 유료 컨텐츠 판매를 통한 금전적인 이득을 얻는 직접적인 수익 창출 금지 (후원, 기부 등은 허용)
    - 소스 코드 및 내부 포함 파일 목록은 의무적으로 완전히 공개 (오픈 소스 강제)
    - 마켓플레이스 컨텐츠의 불법 복제 및 라이선스 침해 금지
    - 유통 플랫폼에 한해 콘텐츠 업로드 단계에서 EULA 동의 과정이 필수적으로 포함
- 데이터 저장
  - 청크 저장 → 차원 별 LevelDB 데이터베이스 파일
    - 렌더 거리 → 최대 96청크
      - 싱글 스레드 메인 틱 연산
    - 시뮬레이션 거리 → 최대 128청크 (일관성 보장, 속도 저하)
      - 멀티 스레드 비동기 틱 연산 (일관성 보장 불가, 속도 증가)
    - LOD 가상 청크 → 최대 512청크
      - 틱 연산 없음
  - NBT → SQLite DB & JSON 칼럼
    - SQL 쿼리 능력 활용
    - 핵심 필드 → SQL 컬링 저장
      - UUID, 좌표, 타입 ID, 소유자 등
    - 상태 데이터 → JSON 직렬화 후 TEXT / BLOB 칼럼 저장
      - 체력, 허기, 커스텀 이름, 마법 부여, 아이템, 인벤토리 슬롯 등
    - NBT로 통합 → 아이템 구성 요소 & 텍스트 구성 요소, 월드 데이터
  - 블록 데이터 → 구조체 저장 (개별 객체 X)
- 변수
  - Scoreboard → 지역 변수로 대체
  - Storage → 전역 변수로 대체
  - 매크로 기능 (`$`) → 명령어 중간 변수 참조로 대체
    - 인게임 명령어에서도 가능
    - 파이프라인 사용 시 `$_` 사용 가능
    - 예시 1:
      - `/variable set entity_name string:"zombie"`
      - `/kill @e[type=$entity_name]`
    - 예시 2:
      - `/execute as @e[type=villager] at @s if entity $_ @e[type=villager,distance=..4,tag=virus] run random value 1..1000 | variable set random_value int:$_ } else { variable set random_value int:0`
      - `/execute as @e[type=villager,tag=!virus] if variable $random_value=1..35 run tag add virus`
- 멀티플레이
  - 멀티플레이 대상 → Xbox 친구 추가
    - 포트포워딩 / 외부 서비스 불필요
    - 호스트 (서버 주인) 컴퓨터 자원 이용
    - PC / 콘솔 / 모바일 크로스 플랫폼 지원
  - 서버 구동 방식 → 별도 인스턴스
    - 로컬 호스트 X
    - 호스트 게임 종료하더라도 서버는 유지
  - 동기화 네트워크 프로토콜 → RakNet
  - 서버 관리 방법 → 구동 인스턴스 GUI 창
    - 2D 월드 관전
    - 서버 규칙 변경
    - 서버 공지
    - CLI 커맨드 콘솔
  - 권한 체계 → 5단계 (0~4)
    - Lv.0 → 참여자 (Guest)
    - Lv.1 → 사용자 (User)
      - 신규 플레이어 초기 권한의 기본값
    - Lv.2 → 운영자 (Operator)
      - 신규 플레이어 초기 권한의 임계값
    - Lv.3 → 관리자 (Administrator)
      - 서버 소유 플레이어 기본 권한
      - 콘솔에서만 부여 가능
    - Lv.4 → 소유자 (Owner)
      - 서버 인스턴스 권한
      - 인게임 플레이어 획득 불가능
- 서비스
  - 무료 마이그레이션 → 월드 파일
    - Java Edition & Bedrock Edition → NEO
    - 웹 사이트 형식
    - Xbox 계정 로그인 필수
  - 베타테스터 헤택
    - 베타 버전 24시간 이상 플레이 + 버그 리포트 / 건의 사항 남길 시 전용 망토 지급

    - 최대 수용 인원 20명 이외에는 제한 사항 없음
